\subsection{Prerequisites}
\begin{itemize}
  \item Rust toolchain (\texttt{cargo}) for building or using the crate.
  \item Python 3 in \texttt{PATH} for rendering \texttt{.v} and \texttt{.inst} from generated Python.
\end{itemize}

PyTV invokes \texttt{python3} on Linux/macOS and \texttt{python} on Windows.

\subsection{Install as a Rust Library}
Add PyTV to an existing Rust project:
\begin{verbatim}
cargo add pytv
\end{verbatim}

This updates \texttt{Cargo.toml} with a dependency entry similar to:
\begin{verbatim}
[dependencies]
pytv = "0.6.0"
\end{verbatim}

Default crate features include \texttt{inst} and \texttt{macro}.
If needed, enable or disable them explicitly in \texttt{Cargo.toml}.

\subsection{Install as a Binary Tool}
Install from crates.io:
\begin{verbatim}
cargo install pytv
\end{verbatim}

Build from source:
\begin{verbatim}
cargo build --release
\end{verbatim}
The compiled binary is available at \texttt{target/release/pytv}.

\subsection{Validate the Installation}
After installation, verify command-line availability:
\begin{verbatim}
pytv --help
\end{verbatim}

\subsection{Install Python Binding (\texttt{tverilog})}
The Python binding is provided in this repository under \texttt{tverilog/}.
It requires Python packaging tools and a Rust toolchain to build the native bridge.

A source install workflow is:
\begin{verbatim}
python -m pip install --upgrade pip maturin
maturin build -m tverilog/native/Cargo.toml \
  --release --interpreter python
python -m pip install --force-reinstall \
  tverilog/native/target/wheels/*.whl
python -m pip install -e tverilog
\end{verbatim}

Details for API usage are described in Section~\ref{sec:tverilog}.
